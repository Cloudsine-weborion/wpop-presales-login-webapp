SQL Injection

Overview

SQL injection is a web application vulnerability as part of the OWASP Top Ten 2021. It is being exploited through SQL commands to search and extract sensitive information

Juice Shop Scenario

Exploiting this vulnerability on juice-shop to retrieve the userâ€™s login email and password:

1. Discovered that there is a /search endpoint that allows users to retrieve product information

2. This discovery proves that there is an endpoint that allows access to the database. Inspect the network traffic and you can find out that the endpoint is /rest/products/search

3. Since this endpoint fetches the product information from the PRODUCTS table. We can craft a UNION select payload to query from the USERS table

4. UNION select works in a way where both table columns need to match. Therefore, we need to first discover how many columns are there in the PRODUCTS table 

      
        /rest/products/search?q=test%27%29%29%20UNION%20SELECT%201%20FROM%20USERS--

5. Do this repeatedly until we find out that the number of columns for the PRODUCTS table is 9

6. Therefore, now we can craft the sql injection payload targeting the USERS table email and password using UNION select 

      
    /rest/products/search?q=qwert%27%29%29%20UNION%20SELECT%20id%2C%20email%2C%20password%2C%20%274%27%2C%20%275%27%2C%20%276%27%2C%20%277%27%2C%20%278%27%2C%20%279%27%20FROM%20Users--
